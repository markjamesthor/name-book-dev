# BlazePose 모델 로컬 저장 가이드

## 현재 상황

TensorFlow.js의 `pose-detection` 라이브러리는 내부적으로 모델 URL을 관리하므로, 
직접 모델 파일 경로를 지정하기 어렵습니다.

## 해결 방법

### 방법 1: 브라우저 캐시 활용 (권장)
- 모델은 첫 로드 시 한 번만 다운로드됩니다
- 브라우저 캐시에 자동 저장되어 다음 로드부터는 빠르게 로드됩니다
- **추가 작업 불필요** - 이미 작동 중입니다

### 방법 2: 완전 오프라인 사용 (고급)

완전히 오프라인으로 사용하려면:

1. **모델 파일 수동 다운로드**
   - 브라우저 개발자 도구에서 네트워크 탭 확인
   - 모델 로드 시 다운로드되는 파일들을 확인
   - 해당 파일들을 `models/blazepose-heavy/` 폴더에 저장

2. **서버에서 모델 파일 제공**
   - 서버가 이미 정적 파일을 제공하도록 설정되어 있습니다
   - `models/` 폴더의 파일들이 자동으로 서빙됩니다

3. **모델 로더 설정 변경** (라이브러리 지원 시)
   - `pose-detection` 라이브러리가 로컬 모델을 지원하는 경우
   - 코드에서 모델 경로를 지정

## 현재 설정

현재 코드는 CDN에서 모델을 로드하지만, 브라우저 캐시 덕분에:
- ✅ 첫 로드: 모델 다운로드 (한 번만)
- ✅ 이후 로드: 캐시에서 빠르게 로드

## 모델 크기

BlazePose Heavy 모델은 약 10-20MB 정도입니다.
첫 로드 후에는 캐시에서 즉시 로드되므로 추가 다운로드가 없습니다.

## 권장 사항

**현재 설정을 그대로 사용하는 것을 권장합니다.**
- 브라우저 캐시가 자동으로 작동합니다
- 추가 설정이 필요 없습니다
- 네트워크가 있는 환경에서는 문제없이 작동합니다

완전 오프라인 사용이 필요한 경우에만 추가 설정을 고려하세요.
